Setting Up Tether

Install from the GitHub repo, connect it to your project, and run your first bootstrap in under 10 minutes.

Python 3.11+ Not on PyPI yet Claude Haiku or Ollama ~5 min setup

Prerequisites

You need four things before installing. All are free except the Anthropic API key (which you can skip if you use Ollama instead).

Python
3.11 or newer. Check with:
python --version
Git
To clone the repo. Check with:
git --version
pip or uv
For editable installs. uv is faster but optional.
pip --version or uv --version
Anthropic API key
Used for Haiku calls. Free tier available. Or use Ollama instead — see below.
platform.anthropic.com/keys
ℹ
No PyPI package yet. Tether is in active development. Install it as an editable local package directly from the cloned repo. When it reaches PyPI you'll be able to do pip install tether-engine — for now follow the steps below.

Install from source

Clone the repo and install it as an editable package. This gives you the tether CLI command and lets you pull updates with a simple git pull.

# 1. Clone the repository
$ git clone https://github.com/kiranktb98/project-tether.git
$ cd project-tether
 
# 2a. Install with pip (standard)
$ pip install -e .
 
# 2b. Or install with uv (faster, recommended)
$ uv pip install -e .
 
# 3. Confirm the CLI is available
$ tether --version
tether, version 0.8.0
✓
Editable install means live updates. Any time you do git pull inside the cloned folder, you get the latest version immediately — no reinstall needed.

Installing Ollama support (optional)

If you want to use a local model instead of the Anthropic API, install the optional httpx dependency:

$ pip install -e ".[ollama]"
# installs httpx for Ollama HTTP calls

API key

Tether uses Claude Haiku for impact analysis and edge case generation. You need an Anthropic API key. Skip this section if you're going the Ollama route.

Environment variable
Project .env file
# macOS / Linux — add to ~/.zshrc or ~/.bashrc for persistence
$ export ANTHROPIC_API_KEY="sk-ant-..."
 
# Windows PowerShell — add to $PROFILE for persistence
> $env:ANTHROPIC_API_KEY = "sk-ant-..."

Tether automatically loads a .env file from the project root on startup.

# Create a .env file in your project root
$ echo 'ANTHROPIC_API_KEY=sk-ant-...' > .env
 
# Make sure .env is in your .gitignore!
$ echo '.env' >> .gitignore
⚠
Never commit your API key. Always add .env to .gitignore before creating it.

tether init

tether init creates the .tether/ directory, writes a default config, and injects tether's workflow rules into your CLAUDE.md. Run it once per project, from the project root.

# Navigate to your project root first
$ cd /path/to/your/project
$ tether init
 
Project name [my-project]: my-project
Language [python]: python
Watcher provider [anthropic]: anthropic
Budget per session USD [5.00]: 5.00
 
✓ Created .tether/config.yaml
✓ Created .tether/
✓ Updated CLAUDE.md with tether workflow rules

What init creates

.tether/
  ├── config.yaml ← project settings (provider, budget, paths)
  ├── ledger.yaml ← feature ledger (populated by bootstrap)
  ├── PLAN.md ← impact analysis output (written by tether plan)
  ├── DRIFT.md ← drift alerts (written by tether watch)
  └── tests/ ← generated edge-case tests (one dir per feature)

CLAUDE.md integration

Init writes a managed block into CLAUDE.md that teaches Claude Code when to run tether and what to do with the output. This is how tether integrates with Claude Code — no hook setup, no extensions needed.

ℹ
Already have a CLAUDE.md? The block is injected at the top with start/end markers, so your existing content is preserved. Running tether init again safely replaces only the managed block.

tether bootstrap

Bootstrap scans your codebase, proposes an initial feature ledger, and generates edge cases for each feature. Run it once after init. It makes LLM calls — expect 2–5 minutes on a typical project.

$ tether bootstrap
 
Scanning codebase...
Found 12 Python files, 4 directories
Proposing features (chunk 1/2)...
Proposing features (chunk 2/2)...
Merging proposals → 6 features
 
────────────────────────────────────────────────────
Review the proposed ledger in your editor.
Edit freely — delete duplicates, add missing features.
Save and close when done.
────────────────────────────────────────────────────
 
[editor opens .tether/ledger.yaml for review]
 
Generating edge cases for f1 (rate limiting)...
Generating edge cases for f2 (auth)...
...
 
Bootstrap complete. Ledger version 1 written.
Cost: $1.42 · 6 features · 41 edge cases

The review step is important

Bootstrap opens your text editor with the proposed ledger before generating edge cases. This is the most important step — the quality of everything tether does downstream depends on the ledger being accurate.

What to do in the editor

  • Delete features that aren't real user-facing features
  • Merge duplicates (Haiku sometimes splits one feature in two)
  • Add features tether missed (ones with no files yet)
  • Fix file paths that are wrong
  • Rename features to match your own terminology

What not to worry about

  • Edge cases — tether generates those after you save
  • History — gets populated as you use tether
  • Completeness — you can always add features later
  • Perfect descriptions — one sentence is fine
⚠
Don't rubber-stamp. If you just save without reviewing, the ledger will contain whatever Haiku guessed. Impact analysis is only as good as the ledger. Spend 5–10 minutes here — it pays for itself immediately.

Skip the editor with --yes

For testing or when you want to get bootstrap done quickly and fix the ledger manually later:

$ tether bootstrap --yes
Accepting proposed ledger without review...

Use Ollama (free, no API key)

Tether supports Ollama as a drop-in replacement for Claude Haiku. You run Ollama locally — no API key, no cost, no data sent externally.

1. Install Ollama and pull a model

# Install Ollama from ollama.ai, then:
$ ollama pull qwen2.5-coder:7b
pulling manifest... done
 
# Start the Ollama server (keep this running)
$ ollama serve
Ollama is running on http://localhost:11434
ℹ
Recommended models. qwen2.5-coder:7b works well for impact analysis and edge case generation. mistral:7b and llama3.1:8b are also good options. Larger models produce better results but run slower.

2. Configure tether to use Ollama

Edit .tether/config.yaml in your project:

# .tether/config.yaml
watcher:
provider: ollama # ← change from anthropic
model: qwen2.5-coder:7b # ← your Ollama model
ollama_base_url: http://localhost:11434 # ← default, change if different

No API key needed. Run tether bootstrap as normal — it will call your local model instead.

⚠
Ollama quality vs Haiku. Local models produce somewhat lower-quality impact analysis and edge case generation than Claude Haiku, especially on nuanced reasoning. For trying out tether or working offline, Ollama is great. For a production project, Haiku is recommended.

Using on your own project

The demo walkthrough above uses the examples/ratelimiter/ sample project. Here's how to set tether up on your actual codebase.

1

Navigate to your project root

$ cd /path/to/your-project
# This is the directory that contains your src/ or main Python files
2

Run init and bootstrap

$ tether init
$ tether bootstrap
3

Start the watcher alongside Claude Code

Open two terminal windows. In the first, start tether's file watcher. In the second, start Claude Code as normal.

# Terminal 1 — keep this running
$ tether watch
Watching /your-project for changes...
 
# Terminal 2 — your Claude Code session
$ claude

Claude Code reads CLAUDE.md at the start of every session. It will automatically run tether plan before changes and check .tether/DRIFT.md for drift alerts.

What was created

After init + bootstrap, your project has a new .tether/ directory. Here's what each file does:

FileCreated byPurpose
.tether/config.yaml tether init Provider, model, budget, paths — edit to change settings
.tether/ledger.yaml tether bootstrap The feature ledger. Source of truth for features, files, edge cases, history
.tether/PLAN.md tether plan Impact analysis for the most recent planned change. Claude Code reads this
.tether/DRIFT.md tether watch Drift alerts. Written when a file change breaks a feature test. Claude Code reads this
.tether/tests/feature_f1/ tether bootstrap Generated pytest tests for each feature's must-handle edge cases
.tether/events.jsonl tether watch / plan JSONL log of every model call with cost tracking
.tether/reports/ tether report Session summaries — run tether report to generate one
CLAUDE.md (block) tether init Workflow rules injected for Claude Code — when to run tether and what to do
⚠
Add .tether/ to .gitignore. The ledger and events log are local state — they should not be committed. Run echo '.tether/' >> .gitignore if tether init didn't do this already.

Cost expectations

Tether uses Claude Haiku for all model calls. Haiku is the cheapest Anthropic model ($1/M input tokens, $5/M output tokens). Here's what to expect:

🔧
tether bootstrap — one-time cost
~$0.80–2.00 depending on project size. A 10-feature project costs ~$0.80. A 30-feature project with detailed edge cases costs ~$1.50–2.00. This is a one-time cost.
📋
tether plan — per change
$0.003–0.02 per run. Depends on how many features are affected by the file you're changing and how many edge cases they have. Average: ~$0.006.
👁
tether watch — per file change
$0.002–0.01 per file change detected (intent check + optional drift attribution). In a typical session of 10–15 file changes, expect $0.05–0.15 for watch alone.
💰
Total per session
A typical Claude Code session with 15–20 changes runs $0.20–0.50 of Haiku overhead. The default budget cap is $5.00 per session — tether stops making model calls if this is exceeded.
ℹ
Ollama = $0. If you use the Ollama provider, all costs are $0.00 — you're running a local model. The tradeoff is somewhat lower quality in impact analysis and edge case generation.

Troubleshooting

command not found: tether

The package wasn't installed into a PATH-visible location. Try:

# Make sure you installed from the cloned directory
$ cd project-tether && pip install -e .
 
# If using a virtualenv, activate it first
$ source .venv/bin/activate # or .venv\Scripts\activate on Windows
$ pip install -e /path/to/project-tether

Error: ANTHROPIC_API_KEY is not set

The API key isn't in your environment. Either export it or add it to a .env file at the project root. See the API key section above.

No features in ledger

Bootstrap hasn't been run yet, or the ledger file is empty. Run tether bootstrap from the project root.

bootstrap already running (PID ...)

A previous bootstrap was interrupted and left a lock file behind. If no bootstrap is actually running:

$ rm .tether/bootstrap.lock
$ tether bootstrap

OllamaConnectionError

Tether is configured to use Ollama but the server isn't running. Start it with ollama serve in a separate terminal.

The editor doesn't open during bootstrap

Tether uses the $EDITOR environment variable to open the draft ledger. Set it if it's not set:

$ export EDITOR=nano # or vim, code, etc.
$ tether bootstrap
 
# Or skip the editor entirely
$ tether bootstrap --yes
